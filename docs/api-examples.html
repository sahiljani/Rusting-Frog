Authentication

Laravel (or any upstream) mints a short-lived HS256 JWT with tenant_id, user_id, and scopes claims, signed with the shared JWT_SECRET. Every API call passes it as Authorization: Bearer <token>. For local testing, scripts/mint_jwt.js mints one signed with the dev secret in .env.

SHELL node scripts/mint_jwt.js

Dev-only helper. Prints a JWT valid for 1 hour with tenant test-tenant.

Command
JWT=$(node scripts/mint_jwt.js | tail -1)
echo $JWT

Catalog

Static registry the frontend renders tabs and filter dropdowns from. Tabs and filters are immutable at runtime — this endpoint is cache-friendly.

GET /v1/catalog/tabs 200

Lists all 33 SF-style tabs with their filter sets. Deprecated filters are omitted. Total payload: 33 tabs × up to ~400 filters each (AI tab is largest, ~401 entries: 1 aggregator + 4 providers × 100 slots).

Projects

A project is a named "site" with a persisted seed URL and config. All crawls are children of a project.

POST /v1/projects 200

Create a new project. Returns the project row immediately.

Request
curl -X POST \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{"name":"boltrack","seed_url":"https://boltrack.in"}' \
  http://127.0.0.1:3000/v1/projects

Crawl lifecycle

A crawl is one execution of the spider against a project's seed URL. POST enqueues the job to Redis — the worker picks it up and starts fetching. Status transitions: queued → running → completed / failed, with paused and stopped as optional terminal states.

POST /v1/projects/{project_id}/crawls 200

Start a crawl. Body is optional — if omitted, uses the project's stored seed URL.

Request
curl -X POST \
  -H "Authorization: Bearer $JWT" \
  -H "Content-Type: application/json" \
  -d '{}' \
  http://127.0.0.1:3000/v1/projects/019d98d7-6896-7a73-ac63-b0315adf6af6/crawls
Response
{
  "id": "019d98d7-a11a-74f3-ae24-d605bc206dda",
  "status": "queued",
  "message": "crawl queued"
}

POST /v1/crawls/{id}/pause

Idempotent pause signal. Worker drains its in-flight URLs and persists frontier.

POST /v1/crawls/{id}/resume

Resume a paused crawl. Frontier is restored from the DB.

POST /v1/crawls/{id}/stop

Terminate permanently. Status transitions to completed (if any URLs were processed) or failed.

The hot path — the React grid hits these constantly. All list endpoints are cursor-paginated. Filter keys are the stable snake_case identifiers from /v1/catalog/tabs.

Error envelope

All errors are JSON. 4xx carry a short error and message; 5xx degrade to {"error":"internal"} — detail is logged server-side with the trace id.

401 Unauthorized

{ "error": "unauthorized", "message": "missing or invalid bearer token" }

404 Not Found

{ "error": "not_found", "message": "crawl 019d98d7-… not found" }

422 Unprocessable

{ "error": "validation", "message": "seed_url must be http(s)" }

FilterKey — on-wire shape

Every filter key in request/response bodies is the same snake_case identifier. The four user-configurable families are parameterized by slot (and, for AI, by provider):

Shape
// Built-in filters (457 total, one per SF filter)
"internal_html"
"title_over_x_characters"
"response_code_client_error"
"hreflang_missing_self_reference"

// User-configurable slots (100 each)
"custom_search_slot_1"    …  "custom_search_slot_100"
"custom_extractor_slot_1" …  "custom_extractor_slot_100"
"custom_javascript_slot_1"…  "custom_javascript_slot_100"

// AI extraction slots: 4 providers × 100 = 400
"ai_openai_slot_1"      …  "ai_openai_slot_100"
"ai_anthropic_slot_1"   …  "ai_anthropic_slot_100"
"ai_gemini_slot_1"      …  "ai_gemini_slot_100"
"ai_ollama_slot_1"      …  "ai_ollama_slot_100"

// Aggregators
"custom_search_all"
"custom_extractor_all"
"custom_javascript_all"
"ai_all"

On the Rust side these are a single FilterKey enum with 457 fixed variants and 4 parameterized variants (CustomSearchSlot(u8), CustomExtractorSlot(u8), CustomJavaScriptSlot(u8), AiSlot(AiProvider, u8)). Slot numbers are validated to 1..=100 on deserialize — out-of-range slots return 422.
